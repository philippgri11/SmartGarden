# Codex Arbeitsregel

Bei groesseren oder fuer Nutzer sichtbaren Aenderungen muss `VERSION.md` gepflegt werden.

Die Datei ist die Grundlage fuer die automatische Changelog-Mail. Schreibe dort so, dass es auch jemand ohne GitHub- oder Programmierwissen versteht:

- Was ist neu?
- Was wurde verbessert?
- Muss jemand etwas tun?

Keine Passwoerter, Tokens oder anderen Geheimnisse eintragen.
